# CCPC 당회록 검색 시스템 — 설치 가이드

구글 드라이브 여러 폴더에 흩어져 있는 당회록(.docx) 파일들을
키워드로 검색할 수 있는 웹 검색 도구입니다.

구조는 기존 CCPC 프로젝트들과 동일합니다:
- **Google Sheets** → 검색용 색인(캐시) 데이터베이스
- **Google Apps Script** → 드라이브 탐색 + .docx 파싱 + 검색 API (JSONP)
- **GitHub Pages** → 검색 화면 (index.html)

---

## 1단계 — 색인용 구글 시트 만들기

1. 새 구글 시트를 하나 만듭니다. (이름 예: `당회록 검색 DB`)
2. 시트 자체에는 아무것도 입력하지 않아도 됩니다. 스크립트가 자동으로
   `당회록색인` 탭과 헤더를 만듭니다.

## 2단계 — Apps Script 코드 붙여넣기

1. 시트에서 **확장 프로그램 → Apps Script** 클릭
2. 기본 `Code.gs` 내용을 모두 지우고, 첨부된 `Code.gs` 내용을 붙여넣기
3. 코드 상단의 설정값은 이미 입력되어 있습니다:

   ```javascript
   const ROOT_FOLDER_ID = '1VvI1kb6PM91eza1EiU_KyMNl2WKGvOMT'; // 당회록 최상위 폴더 (이미 설정됨)
   ```

   이 폴더 아래의 모든 하위 폴더(연도별/월별로 나뉘어 있어도 무관)를
   자동으로 재귀 탐색합니다. 폴더를 옮기거나 바꾸게 되면 이 값만
   교체하면 됩니다.

4. 파일명에 "당회록"이라는 글자가 들어있는 .docx만 자동으로 인식합니다.
   (`FILENAME_FILTER` 값으로 변경 가능)

## 3단계 — Drive API 서비스 추가 (필수)

.docx 파일의 텍스트를 읽으려면 임시로 구글 문서로 변환하는 과정이 필요한데,
이를 위해 고급 Drive API 서비스를 켜야 합니다.

1. Apps Script 편집기 좌측의 **서비스 (Services)** 옆 `+` 클릭
2. 목록에서 **Drive API** 선택 → 추가
   (참고: 식별자는 `Drive` 로 자동 지정됩니다)

## 4단계 — 첫 색인 실행

1. 함수 선택 드롭다운에서 `rebuildIndex` 선택
2. ▶ 실행 버튼 클릭
3. 처음 실행 시 권한 승인 화면이 나옵니다 → 본인 계정으로 승인
4. 실행이 끝나면 시트의 `당회록색인` 탭에 당회록들이 한 줄씩 채워진 것을
   확인할 수 있습니다.

> **참고 (파일이 많은 경우):** Apps Script는 한 번 실행에 6분 제한이 있습니다.
> 당회록이 아주 많다면 `MAX_FILES_PER_RUN` 값만큼만 처리하고 멈추도록 되어
> 있습니다. 이 경우 `rebuildIndex`를 몇 번 더 실행하거나, 5단계의 자동
> 트리거를 걸어두면 시간이 지나며 자연히 전체가 색인됩니다.

## 5단계 — 자동 갱신 트리거 설정 (권장)

새 당회록이 매달 추가될 것이므로, 자동으로 색인이 갱신되도록 설정합니다.

1. Apps Script 좌측의 **트리거(시계 아이콘)** 클릭
2. **트리거 추가**
   - 실행할 함수: `rebuildIndex`
   - 이벤트 소스: 시간 기반
   - 시간 기반 트리거 유형: 일 단위 타이머
   - 시간대: 새벽 시간대 권장 (예: 오전 2~3시)
3. 저장

이미 색인된 파일은 수정되지 않았으면 다시 파싱하지 않으므로, 매일 실행해도
새로 추가되거나 수정된 당회록만 빠르게 갱신됩니다.

## 6단계 — 웹 앱으로 배포

1. Apps Script 우측 상단 **배포 → 새 배포**
2. 유형 선택(⚙️) → **웹 앱**
3. 설정:
   - 설명: `당회록 검색 API` (자유롭게)
   - 실행 계정: **나**
   - 액세스 권한이 있는 사용자: **모든 사용자**
4. **배포** 클릭 → 권한 승인
5. 나오는 **웹 앱 URL**을 복사해 둡니다.
   (예: `https://script.google.com/macros/s/AKfycb.../exec`)

> 코드를 수정한 뒤에는 "새 배포"가 아니라 기존 배포의 **편집(연필 아이콘) →
> 새 버전**으로 업데이트해야 동일한 URL이 유지됩니다.

## 7단계 — GitHub Pages 프론트엔드 연결

화면은 **탭 두 개**로 구성되어 있습니다: "📄 새 당회록 작성" (템플릿 다운로드 +
드라이브 폴더 바로가기) 과 "🔍 당회록 검색" (키워드 검색). 같은 화면 안에서
탭만 눌러서 전환합니다.

1. 첨부된 `index.html` 파일을 엽니다.
2. 아래 줄을 찾아서 6단계에서 복사한 웹 앱 URL로 교체합니다:

   ```javascript
   const API_URL = 'PUT_APPS_SCRIPT_WEB_APP_URL_HERE';
   ```

3. `glenwercpc-byte` GitHub 조직 아래 새 저장소(예: `Session-Minutes`)를 만들고
   다음 3개 파일을 **모두 같은 폴더에** 업로드합니다:
   - `index.html`
   - `당회록_템플릿.docx` ← "템플릿 다운로드" 버튼이 이 파일을 가리킵니다.
     (파일명을 바꾸면 index.html 안의 `href="당회록_템플릿.docx"` 부분도
     같이 바꿔주세요.)
4. 저장소 **Settings → Pages** 에서 GitHub Pages를 활성화합니다.
5. 발급된 주소(예: `https://glenwercpc-byte.github.io/저장소이름/`)로 접속하면
   템플릿/검색 화면이 보입니다.

> "구글 드라이브 폴더 열기" 버튼은 이미 실제 당회록 폴더
> (`1VvI1kb6PM91eza1EiU_KyMNl2WKGvOMT`)로 연결되어 있어 별도 설정이
> 필요 없습니다.

---

## 사용 방법

**📄 새 당회록 작성 탭**: 템플릿을 다운로드하여 작성한 뒤, 같은 탭의
"구글 드라이브 폴더 열기" 버튼으로 이동해 완성된 파일을 업로드합니다.

**🔍 당회록 검색 탭**: 검색창에 키워드를 입력하면 **논의 사항 / 결의 사항**
본문에서 해당 단어가 포함된 모든 당회록을 찾아, 일시·장소·참석자와 함께
일치하는 부분을 강조해서 보여줍니다. 각 결과 카드의 "원본 문서 보기" 버튼을
누르면 구글 드라이브에 저장된 실제 .docx 파일이 새 탭에서 열립니다.

## 향후 확장 아이디어 (필요시 요청해 주세요)

- 날짜 범위 검색 / 참석자 이름 검색 필터 추가
- 결의 사항에 번호를 매겨 "결의 사항 #3 검색" 같은 정확 매칭
- 검색 결과를 엑셀로 내보내기
- 매번 전체 재탐색이 아니라, 새 파일 추가 시 알림(Drive 변경 트리거)으로
  즉시 색인되도록 개선
